## Overview

This automation framework is built using Playwright and follows the Page Object Model (POM) design pattern to ensure maintainable and scalable test automation.

### Key Components

- **Pages/**: Contains page object classes that encapsulate the functionality of each page
- **TestCases/**: Contains test case descriptions and documentation
- **Tests/**: Contains the actual test implementations
- **Utils/**: Contains utility functions and test data
- **Allure-results/**: Generated test reports in Allure format
- **Playwright-report/**: HTML test reports generated by Playwright
- **Test-results/**: Screenshots, videos, and traces from test executions

### Folder Structure Details

- **`.github/`**: Contains GitHub Actions workflows and repository configurations
- **`allure-results/`**: Auto-generated Allure test result files
- **`node_modules/`**: npm dependencies (automatically generated, excluded from version control)
- **`pages/`**: Page Object Model classes for different application pages
- **`playwright-report/`**: HTML reports generated after test execution
- **`TestCases/`**: Test case description files with test names and tags
- **`test-results/`**: Test artifacts including screenshots, videos, and execution traces
- **`tests/`**: Organized test files by feature/functionality
- **`utils/`**: Helper functions, test data generators, and configuration files

### Main Features

- **App Admin Navigation Testing**: Comprehensive testing for App Admin role access to Clear Cache, Cache Statistics, and Site Config
- **Jira Webhook Testing**: Comprehensive testing for Jira webhook creation, management, and operations
- **Authentication Testing**: Tests for user authentication flows
- **Page Object Model**: Organized structure for better maintainability
- **Test Data Management**: Utilities for generating and managing test data

## Getting Started

1. Install dependencies:
   ```bash
   npm install
   ```

2. Configure test environment in `playwright.config.js`

3. Update user credentials in `utils/userCredentials.json`

4. Run tests:
   ```bash
   npx playwright test
   ```

## Environment Variables & Security

### What the `.env` file does
The `.env` file stores sensitive credentials (Jira URL, email, API token) outside of test code so that nothing is hardcoded. These values are loaded automatically at runtime by the `dotenv` package.

### First-Time Setup
1. Copy the example file and rename it:
   ```bash
   cp .env.example .env
   ```
2. Open `.env` and fill in your real values:
   ```
   JIRA_BASE_URL=https://your-domain.atlassian.net
   JIRA_EMAIL=your-email@example.com
   JIRA_API_TOKEN=your_api_token_here
   ```
3. Save the file — no test code needs to be changed.

> **Note:** The Jira connection name is generated dynamically at runtime. Do not add it to `.env`.

### The Two Files — At a Glance

| File | Commit to Git? | Contains Real Values? |
|---|---|---|
| `.env.example` | ✅ Yes — safe to share | ❌ No — empty placeholders only |
| `.env` | ❌ Never | ✅ Yes — your actual secrets |

### Security Rules

- `.env` is already listed in `.gitignore` — **never remove it from there**.
- Never paste `.env` contents in Slack, emails, pull requests, or code comments.
- Share credentials with teammates only through a **secure channel** (e.g. a password manager or secrets vault).
- If a token is accidentally committed or leaked → **revoke it from Atlassian immediately** and generate a new one.
- Do not log credential values in test output or console statements.

> ⚠️ **WARNING:** Committing real API tokens to the repository is a security risk. If this happens, treat the token as compromised — revoke it first, then rotate credentials before continuing.

### When Credentials Change or Expire
- Update only your local `.env` file with the new values.
- `.env.example` stays unchanged — it never holds real values.
- No page files or test files need to be modified.

### How It Works in Code
The `playwright.config.js` loads the `.env` file at startup:
```js
import dotenv from 'dotenv';
import path from 'path';

dotenv.config({ path: path.resolve(__dirname, '.env') });
```
After that, any file in the project can read values via:
```js
process.env.JIRA_BASE_URL
process.env.JIRA_EMAIL
process.env.JIRA_API_TOKEN
```

## Test Execution Commands

### Regression Tests
- **List regression tests**: `npx playwright test --list --grep "@regression"`
- **Run regression tests**: `npx playwright test --workers=1 --grep "@regression"`
- **Run regression tests in headed mode**: `npx playwright test --workers=1 --grep "@regression" --headed`

### Sanity Tests
- **List sanity tests**: `npx playwright test --list --grep "@sanity"`
- **Run sanity tests**: `npx playwright test --workers=1 --grep "@sanity"`
- **Run sanity tests in headed mode**: `npx playwright test --workers=1 --grep "@sanity" --headed`

### App Admin Tests
- **Run App Admin tests**: `npx playwright test tests/App_Admin/`
- **Run specific App Admin test**: `npx playwright test tests/App_Admin/TC01_appAdminCompleteNavigation.spec.js`
- **Run App Admin tests in headed mode**: `npx playwright test tests/App_Admin/ --headed`

### Command Options Explained
- `--workers=1`: Runs tests sequentially (single worker)
- `--grep "@tag"`: Filters tests by tag (e.g., @regression, @sanity)
- `--headed`: Runs tests in headed mode (browser visible)
- `--list`: Lists matching tests without executing them

## Test Categories

- **App Admin Tests**: Verify App Admin navigation and access to admin screens (Clear Cache, Cache Statistics, Site Config)
- **Authentication Tests**: Verify login/logout functionality
- **Jira Webhook Tests**: End-to-end testing of webhook creation and management workflows

## Troubleshooting

If you encounter issues running the tests:

1. **Error: Cannot find package '@playwright/test'**:
   ```bash
   npm install
   npx playwright install
   ```

2. **Install Playwright browsers**:
   ```bash
   npx playwright install
   ```

3. **Check if tests are listed correctly**:
   ```bash
   npx playwright test --list
   ```

4. **Verify specific tag tests**:
   ```bash
   npx playwright test --list --grep "@sanity|@regression"
   ```

5. **Run a single test file**:
   ```bash
   npx playwright test tests/authentication/TC01_loginTest.spec.js
   npx playwright test tests/App_Admin/TC01_appAdminCompleteNavigation.spec.js
   ```

6. **Debug mode**:
   ```bash
   npx playwright test --debug --grep "@sanity"
   ```
